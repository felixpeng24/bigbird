Raspberry Pi 5 Base Setup

Flash Raspberry Pi OS to the microSD card

On your Mac, download and run the Raspberry Pi Imager:

  1. Download from raspberrypi.com/software
  2. Insert your microSD card into your Mac (use an adapter if needed)
  3. Open Raspberry Pi Imager
  4. Click Choose Device → select Raspberry Pi 5
  5. Click Choose OS → select Raspberry Pi OS (64-bit) (the full desktop version, not Lite)
  6. Click Choose Storage → select your microSD card
  7. Click Next, then Edit Settings

In the settings dialog, configure:

  • Hostname: bigbird
  • Username: pi
  • Password: something you'll remember
  • WiFi: enter your WiFi network name and password
  • Locale: set your timezone and keyboard layout
  • Under Services tab: enable SSH (password authentication)

Click Save, then Yes to flash. Wait for it to finish (~5 min).

Why SSH? Once the Pi is inside the puppet, you'll want to access it remotely from your Mac. SSH lets you do that.

Insert the SD card and do a bare-board boot test

Before attaching anything else, verify the Pi boots:

  1. Insert the microSD card into the Pi 5's card slot (on the bottom of the board)
  2. Connect the Cable Matters Micro HDMI cable from the Pi's HDMI 0 port (the one closest to the USB-C power port) to any monitor or TV you have
  3. Connect a USB keyboard and mouse to the Pi's USB ports
  4. Connect the 27W USB-C power supply to the Pi's USB-C port
  5. The Pi should boot to the Raspberry Pi OS desktop within 30-60 seconds
Don't use the GeeekPi display yet. Use any regular monitor/TV for this initial setup. We'll configure the 7" display resolution later.

Once you see the desktop, open a terminal and run:

sudo apt update && sudo apt upgrade -y

This ensures you have the latest system packages. Takes ~5-10 minutes.

Verify WiFi and SSH work

On the Pi terminal, check your IP address:

hostname -I

You should see something like 192.168.1.42. Write this down.

Now from your Mac, open Terminal and SSH in:

ssh pi@bigbird.local

If .local doesn't resolve, use the IP address directly:

ssh pi@192.168.1.42

Accept the fingerprint prompt, enter your password. You should see a Pi terminal prompt.

✓ Checkpoint: Pi boots, desktop loads, WiFi connected, SSH works from your Mac. You can now shut down the Pi (sudo shutdown now) and move to hardware assembly.

Mount the Active Cooler

Attach the iUniker ICE Peak cooler to the Pi 5

⚡ Make sure the Pi is unplugged and powered off before touching the board.
  1. Peel the release paper from the thermal pad on the bottom of the cooler (the side that touches the CPU chip)
  2. Align the cooler over the Pi 5's CPU (the large silver square chip in the center of the board). The screw holes on the cooler should line up with the two mounting holes on the Pi board.
  3. Press down gently so the thermal pad makes full contact with the CPU
  4. Screw in both mounting screws (included with the cooler) using a small Phillips screwdriver. Tighten snugly — don't overtighten.
  5. Connect the fan cable to the Pi 5's 4-pin JST fan header. This is the small white connector on the upper-right area of the board, between the GPIO pins and the USB 2.0 ports. The cable only fits one way.
Why this matters: The Pi 5 inside a sealed plush puppet will overheat without active cooling. This cooler keeps the CPU under 56°C even under stress. The fan auto-adjusts speed via PWM — you won't need to configure anything.

Test the cooler works

Power on the Pi. You should hear (or feel) the fan spin briefly on boot.

Open a terminal and check the temperature:

vcgencmd measure_temp

Should show something like temp=38.5'C. Anything under 55°C at idle is great.

Verify no throttling:

vcgencmd get_throttled

Should show throttled=0x0 (no throttling).

✓ Checkpoint: Cooler mounted, fan spins, temperature under control.

Connect the Camera

Connect the Waveshare CSI cable to the Pi 5

⚡ Power off the Pi before connecting ribbon cables. The CSI connector is fragile.

The Pi 5 has two CSI/DSI connectors (22-pin, labeled CAM/DISP 0 and CAM/DISP 1). Use CAM/DISP 0 (the one closer to the HDMI ports).

  1. Gently lift the black plastic latch on the CAM/DISP 0 connector by pulling it straight up. It hinges open about 90°. Don't force it.
  2. Slide the 22-pin end of the Waveshare cable into the connector. The metal contacts face the board (toward the Pi's PCB). The blue reinforcement strip faces up.
  3. Push the latch back down to lock the cable. Give the cable a very gentle tug — it should not slide out.
Orientation matters. If the cable is in backwards, the camera won't be detected. Metal contacts face the board.

Connect the other end to the Camera Module 3

  1. The Camera Module 3 has a 15-pin connector. Lift its latch the same way.
  2. Slide the 15-pin end of the Waveshare cable in. Metal contacts face the camera's PCB.
  3. Push the latch down to lock.

The cable is 200mm (about 8 inches) — enough to route from the Pi inside the puppet body up to the eye area.

Test the camera

Power on the Pi and open a terminal:

# Quick test — shows camera preview for 5 seconds
libcamera-hello -t 5000

# Capture a test photo
libcamera-still -o test.jpg

# Check the file
ls -la test.jpg

You should see a camera preview window pop up, then a JPEG file saved. If you get an error like "no cameras available", double-check the ribbon cable orientation and that both latches are fully closed.

✓ Checkpoint: Camera detected, preview works, photo captured.

Set Up the 7" Display

Connect the GeeekPi display

  1. Connect the Cable Matters Micro HDMI cable from the Pi's HDMI 0 port to the display's full-size HDMI port
  2. Power the display via its USB-C port. You can use:
    • One of the Pi's USB-A ports (with a USB-A to USB-C cable, often included with the display), OR
    • A USB-A port on the power strip

Configure the 1024×600 resolution

The 7" display uses a non-standard resolution. Edit the boot config:

sudo nano /boot/firmware/config.txt

Add these lines at the bottom of the file:

# GeeekPi 7" 1024x600 display
hdmi_force_hotplug=1
hdmi_group=2
hdmi_mode=87
hdmi_cvt 1024 600 60 3 0 0 0

Save (Ctrl+O, Enter, Ctrl+X) and reboot:

sudo reboot

After reboot, the display should show the full 1024×600 desktop with no black bars or stretching.

✓ Checkpoint: Display shows the Pi desktop at 1024×600, no distortion.

Set Up Audio

Connect the audio chain

The Pi 5 has no 3.5mm headphone jack, so the audio chain is:

Pi USB-A port │ ▼ Plugable USB Audio Adapter (green 3.5mm female jack) │ ▼ Tan QY extension cable (male end plugs into adapter) │ ▼ (female end) Speaker's built-in retractable male plug inserts here
  1. Plug the Plugable USB Audio Adapter into any of the Pi's USB-A ports
  2. Plug the male end of the Tan QY extension cable into the adapter's green 3.5mm jack (not the red mic jack)
  3. Pull out the speaker's built-in retractable 3.5mm male plug from its base
  4. Insert the speaker's male plug into the female end of the extension cable
  5. Turn on the speaker (power button)

Connect speaker charging

The speaker runs on battery but needs continuous charging for multi-hour operation:

  1. Plug the USB-A end of the CableCreation 6" Micro USB cable into a USB-A port on the power strip
  2. Plug the Micro USB end into the speaker's charging port

The speaker can play audio over aux while charging simultaneously — they're separate ports.

Set USB audio as the default output and test

On the Pi, check that the USB adapter was detected:

aplay -l

You should see a "USB Audio Device" listed (usually as card 1 or 2). Then set it as default output:

# List available sinks
pactl list short sinks

# Set USB audio as default (use the name from the list above)
pactl set-default-sink alsa_output.usb-GeneralPlus_USB_Audio_Device-00.analog-stereo

# Set volume to 80%
pactl set-sink-volume @DEFAULT_SINK@ 80%

# Test with a sound
aplay /usr/share/sounds/alsa/Front_Center.wav

You should hear "Front Center" through the speaker.

The speaker has no volume buttons. All volume is controlled from the Pi via software. Set it to 70-80% as a starting point — you can adjust at the venue.
✓ Checkpoint: USB audio adapter detected, sound plays through the speaker.

Wire the Arcade Button

Identify the microswitch terminals

The 100mm arcade button comes with a separate microswitch that snaps into the button housing. The microswitch has three terminals — you only need two:

  • COM (Common) — usually the middle terminal
  • NO (Normally Open) — the terminal that connects to COM when the button is pressed
  • NC (Normally Closed) — not used

The terminals are usually labeled on the microswitch body. If not, use a multimeter on continuity mode: press the button and find which two terminals connect.

Ignore the LED terminals. The button also has 12V LED terminals (larger 6.3mm spade connectors). Leave these completely unwired — we're not using the LED.

Solder wires to the microswitch

🔥 Soldering safety: Work in a ventilated area. The iron tip is 300°C+. Don't touch the tip or freshly soldered joints.
  1. Cut two pieces of hookup wire, about 12-16 inches (30-40cm) each — long enough to route from inside the puppet to the Pi
  2. Strip about 5mm of insulation from each end of both wires
  3. Solder one wire to the NO terminal of the microswitch
  4. Solder the other wire to the COM terminal
  5. Give each joint a gentle tug to make sure it's solid

On the other end of each wire, attach a female Dupont connector from the Elegoo jumper wire pack. The easiest method: cut a female-to-female jumper wire in half and solder/twist-connect it to your button wire.

Connect to the Pi's GPIO pins

Push the female Dupont connectors onto the Pi's GPIO header:

Pi 5 GPIO Header (top view) ┌─────────────────────────────┐ │ (pin 1) 3V3 5V (pin 2) │ │ (pin 3) SDA 5V (pin 4) │ │ (pin 5) SCL GND (pin 6) │ │ (pin 7) GP4 TX (pin 8) │ ┌───▶ │ (pin 9) GND RX (pin 10) │ │ │ (pin 11) GP17 GP18 (pin 12)│ ◀───┐ │ │ (pin 13) GP27 GND (pin 14) │ │ │ │ ... ... │ │ │ └─────────────────────────────┘ │ │ │ COM wire NO wire (from microswitch) (from microswitch)
  • NO wire → GPIO 17 (physical pin 11)
  • COM wire → GND (physical pin 9)
How it works: GPIO 17 has an internal pull-up resistor (enabled by gpiozero). It normally reads HIGH (3.3V). When you press the button, the microswitch shorts NO to COM, pulling GPIO 17 to GND (LOW). The software detects this falling edge as a button press.

Test the button

On the Pi, run a quick test:

python3 -c "
from gpiozero import Button
from signal import pause

btn = Button(17)
btn.when_pressed = lambda: print('PRESSED!')
btn.when_released = lambda: print('released')
print('Press the arcade button...')
pause()
"

Press the big red button. You should see PRESSED! in the terminal. Press Ctrl+C to exit.

✓ Checkpoint: Button press detected on GPIO 17. The physical input works.

Mount the button (optional — can do at venue)

The 100mm button needs an 88mm mounting hole in a flat surface. Options:

  • Wooden craft box — drill an 88mm hole in the lid with a hole saw
  • Thick MDF/plywood — cut a circle with a jigsaw or hole saw
  • Plastic project box — cut with a craft knife (harder with plastic)

Insert the button from the top, thread the locking ring from below, and tighten by hand. The wires route out the back/bottom to the Pi.

You can also skip mounting for now and just hold the button during testing — mount it at the venue.

Puppet Assembly

Open the plush and create the electronics cavity

This is irreversible. Once you cut the plush open, you're committed. Plan your cuts before starting.

The Jay Franco Big Bird plush is sealed (no zipper). You need to cut open a seam:

  1. Find the back seam — look for a stitched line along the puppet's back, from the neck area down to the lower body
  2. Use a seam ripper or small scissors to open about 8-10 inches of the back seam. This is your access point for all electronics.
  3. Remove or displace weighted beads — the plush contains weighted beads. Push them to the extremities (arms, legs, head) or partially remove them to create a cavity in the torso for the Pi + cooler stack, speaker, and cable routing.
  4. You need roughly 170 × 130 × 80 mm of clear space for the Pi + cooler + adapter + cables

Position the camera in the eye area

  1. The Camera Module 3 (25 × 24mm) is tiny. Find a spot at or near one of Big Bird's eyes.
  2. Cut a small hole in the fabric just large enough for the camera lens (about 10mm diameter)
  3. Position the camera behind the hole so the lens looks outward
  4. Secure with hot glue or a small stitch around the camera board (don't cover the lens)
  5. Route the CSI ribbon cable down through the puppet body to the Pi
Camera in Big Bird's eye — this is the surveillance metaphor made literal. Participants look at Big Bird, and Big Bird looks back.

Position the speaker near the mouth area

The speaker (~50 × 50 × 50mm) should go near Big Bird's beak/throat area so the sound appears to come from the puppet:

  1. Nestle the speaker in the upper chest area, driver (mesh side) facing outward
  2. Route the 3.5mm retractable cable and micro USB charging cable down to the body cavity
  3. Make sure the fabric isn't pressing too tightly against the speaker cone — it needs a little air space to project sound

Install the Pi and route all cables

Place the Pi 5 (with cooler mounted) in the torso cavity. Connect everything:

  • CSI cable from camera → Pi CAM/DISP 0
  • USB audio adapter → Pi USB-A port → extension cable → speaker aux plug
  • HDMI cable routes out to the display (mounted externally on the belly or a stand)
  • Button wires route out to wherever the button is mounted
  • Power cable routes out to the 27W PSU (use the AuviPal 90° adapter for a cleaner bend)
  • Speaker Micro USB routes out to the power strip
🌡️ CRITICAL: Cut a vent hole. The active cooler needs airflow. Cut or leave a small opening (2-3cm) in the puppet fabric near the cooler's exhaust side. Without this, hot air recirculates inside the insulated plush and the Pi will throttle.

Mount the display

The 7" display (~165 × 100mm) mounts externally. Options:

  • Belly mount: Cut a rectangular window in Big Bird's belly, mount the display behind it (face out). Secure with hot glue or velcro.
  • Stand mount: Use the included display stands and position it next to the puppet on a table.
  • Neck/chest mount: Hang it from Big Bird's chest like a placard.

The HDMI and power cables route from the display into the puppet body and connect to the Pi.

Close up the puppet

Once everything is positioned and cables are routed cleanly:

  1. Use a needle and thread to loosely stitch the back seam closed — use big stitches so you can reopen if needed
  2. Alternatively, use hot glue along the seam edges (less reversible) or velcro strips (most reversible)
  3. Leave the vent hole open for the cooler
  4. Make sure no cables are kinked, pinched, or under tension
✓ Checkpoint: All hardware is physically installed. Pi, camera, speaker, and display are connected. Cables route cleanly. Vent hole is open.

Software Installation

Clone the repository onto the Pi

SSH into the Pi from your Mac (or use the Pi's terminal directly):

ssh pi@bigbird.local

Clone the project:

cd ~
git clone https://github.com/YOUR_USERNAME/bigbird.git
cd bigbird

(Replace with your actual repo URL, or copy the files over via scp.)

Create the .env file with your OpenAI API key

nano ~/bigbird/.env

Add this single line:

OPENAI_API_KEY=sk-your-actual-key-here

Save and exit (Ctrl+O, Enter, Ctrl+X).

Do not commit the .env file to git. It contains your API key.

Run the automated installer

The install script handles system packages, Python dependencies, and systemd service setup:

cd ~/bigbird
chmod +x install-pi.sh
./install-pi.sh

This takes about 2-5 minutes. It will:

  • Install chromium-browser, unclutter, python3-picamera2
  • Install Python packages from requirements.txt (Flask, Flask-SocketIO, openai, etc.)
  • Copy bigbird.service to systemd and enable auto-start on boot

When it finishes, you should see: [SETUP] ✓ Installation complete!

Do a manual test run before enabling kiosk mode

Before committing to auto-boot, test everything manually:

cd ~/bigbird
python server.py

Open Chromium on the Pi's desktop and navigate to http://localhost:8080.

You should see:

  • The idle screen with the clay swirl background
  • Live camera feed from the Camera Module 3 (your actual face, not a test image!)
  • "BIG BIRD IS WATCHING YOU" title in Comic Sans

Now press the big red arcade button (or spacebar on a connected keyboard). The full evaluation loop should run:

  1. CAPTURING — "Hold still" with camera flash
  2. ANALYZING — demographics appear one by one (from OpenAI)
  3. CALCULATING — green jargon lines scroll with progress bar
  4. REVEALING — Leadership Index score appears
  5. RESETTING — returns to idle

Listen for TTS through the speaker. You should hear Big Bird's robotic espeak voice narrating each phase.

If TTS is silent: Check that the USB audio adapter is set as default output and volume is up. Run pactl set-sink-volume @DEFAULT_SINK@ 80%.

Press Ctrl+C in the terminal to stop the server when done.

✓ Checkpoint: Full loop works on Pi with real camera, real button, real speaker, real OpenAI API.

Reboot and verify auto-start kiosk mode

This is the moment of truth:

sudo reboot

After reboot (~30-45 seconds), the following should happen automatically with no keyboard or mouse interaction:

  1. Pi boots into the desktop
  2. The bigbird.service starts the Flask server
  3. Chromium opens in full-screen kiosk mode — no browser chrome, no address bar, no close button
  4. The cursor is hidden (via unclutter)
  5. The idle screen appears with the live camera feed

Press the arcade button. The full loop should run. If it does — you're done.

To get out of kiosk mode (for debugging): SSH in from your Mac and run sudo systemctl stop bigbird. Or connect a keyboard and press Alt+F4 to close Chromium.

Final Checks & Venue Prep

Power cycle test — unplug and replug

Simulate a power outage by unplugging the power strip entirely. Wait 10 seconds. Plug it back in.

The Pi should boot, the server should start, Chromium should open in kiosk mode, and the idle screen should appear — all automatically. No intervention needed.

This is critical because the installation may lose power at the venue and needs to recover on its own.

Run the soak test (1+ hour)

Let the installation run for at least 1 hour. Trigger an evaluation every 5-10 minutes. Monitor for:

  • Memory leaks — does the app get slower over time?
  • Temperature — SSH in and run vcgencmd measure_temp periodically. Should stay under 65°C.
  • Camera freezing — does the MJPEG stream keep working after many cycles?
  • Audio glitches — does TTS continue working after many cycles?
  • Phase resets cleanly — does it always return to idle correctly?

Monitor from your Mac via SSH:

# Watch temperature
ssh pi@bigbird.local 'watch -n 5 vcgencmd measure_temp'

# Watch server logs
ssh pi@bigbird.local 'journalctl -u bigbird -f'

Readability test — check text size from 3-5 feet

Stand 3-5 feet from the 7" display and verify:

  • The title "BIG BIRD IS WATCHING YOU" is clearly readable
  • Demographic labels and values are readable during the analysis phase
  • The Leadership Index score is the largest and most readable element
  • The jargon text during calculation is readable (it's smaller, which is fine — it's flavor text)
If text is too small: SSH in and edit ~/bigbird/static/style.css. Bump up font sizes. The score (.score-text) should be at least 4rem. The title should be at least 3.5rem.

Troubleshooting

ProblemFix
Display is black / no signalCheck HDMI cable is in HDMI 0 (closest to USB-C). Verify config.txt changes. Try without the hdmi_cvt line first.
Camera not detectedRibbon cable orientation — metal contacts face the board. Make sure both latches are fully closed. Run libcamera-hello.
No sound from speakerCheck pactl list short sinks and set USB audio as default. Make sure the extension cable is in the green jack (not red mic jack). Verify speaker is turned on.
Button press not detectedCheck GPIO 17 (pin 11) and GND (pin 9) connections. Run the test script. Verify solder joints are solid.
Pi overheating / throttlingCheck cooler fan is spinning. Check vcgencmd get_throttled. Make sure the vent hole is open in the puppet fabric. In extreme cases, enlarge the vent hole.
API returns "SUBJECT DEFIES CLASSIFICATION"This is the fallback — either OpenAI refused or there's no internet. Check WiFi with ping google.com. Check the API key in .env.
Kiosk mode doesn't start on bootsudo systemctl status bigbird to see errors. Check journalctl -u bigbird -n 50 for logs. Verify DISPLAY=:0 is set in the service file.
Chromium shows "Restore pages" dialogThe --disable-session-crashed-bubble and --disable-restore-session-state flags in start.sh should prevent this. If it still happens, delete ~/.config/chromium/Default/Preferences.
Speaker dies during eventVerify the Micro USB charging cable is plugged in (power strip USB-A → speaker). The speaker should charge while playing audio.
Evaluation loop hangsCheck server logs: journalctl -u bigbird -f. Most likely an API timeout — the fallback should handle it, but check WiFi stability.
